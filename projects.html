Projets

Voici une vue d’ensemble de mes projets, classés par technologie principale. Les boutons ci-dessous permettent de filtrer par stack. Les boutons Démo / Code sont là pour lier ensuite vers tes URLs locales (Wamp), tes exécutables ou tes dépôts GitHub.

OVH Manager – Dashboard de gestion

PHP · Symfony · MySQL · DataTables

Tableau de bord permettant de centraliser la gestion des domaines, VPS et factures. Utilisation de Symfony, de requêtes Ajax et de tableaux dynamiques pour filtrer et rechercher rapidement les informations.

Architecture : Symfony (MVC) avec entités Doctrine, contrôleurs REST internes, vues Twig, MySQL comme base de données ; front utilisant DataTables pour l’affichage interactif.

Garbadge – Gestion de repas & stocks

Symfony 6.4 · API Platform · JWT · Monolog

API REST pour gérer des repas, des ingrédients et des stocks, avec authentification via JWT et journalisation via Monolog. Pensé pour optimiser la planification de repas et réduire le gaspillage.

Architecture : Backend full API Platform, entités Doctrine reliées, endpoints auto-générés et personnalisés, sécurité par tokens JWT (LexikJWT), logs via Monolog, testés avec Postman.

Démo Code

PlanningApp – Planification automatisée

Symfony · Doctrine · FullCalendar

Application de planification permettant d’assigner des employés, groupes et intérimaires sur des créneaux. Gestion des contraintes métiers et visualisation via un calendrier interactif.

Architecture : Backend Symfony avec logique métier encapsulée dans des services, entités Doctrine pour les ressources, exposition de données au format JSON consommé par FullCalendar côté front.

Démo Code

SteamRest – Bibliothèque de jeux

Java · Spring Boot · React · API REST

Ensemble back-end / front-end pour gérer une bibliothèque de jeux, les achats et les utilisateurs. Le back Spring Boot expose une API REST consommée par un front React.

Architecture : API REST Spring Boot (contrôleurs, services, repository, entités), base de données relationnelle, front React séparé avec appels HTTP via fetch/axios et gestion d’état local.

Démo Code

JCDecaux Vélo’v – Application de stations

C# · WinForms / Java · API externe

Application de bureau récupérant les données des stations vélos en libre-service via l’API JCDecaux : nombre de vélos disponibles, bornes libres, statut de la station, etc.

Architecture : Client lourd qui consomme une API REST externe, désérialisation JSON, affichage des données dans des contrôles graphiques (liste, tableaux, éventuellement carte).

Démo Code

Interface magasin – Front JS pour API de jeux

JavaScript · Svelte / React · API REST

Interface front consommant une API de gestion de jeux (liste, ajout au panier, achats, etc.). Permet de sélectionner un utilisateur, voir les jeux et envoyer des requêtes POST vers l’API.

Architecture : Front JS basé sur composants (Svelte/React), appels API REST pour récupérer les ressources (utilisateurs, jeux, achats) et mise à jour dynamique de l’interface.

Démo Code

Environnements Docker & outils IA

Docker · Scripts · IA

Mise en place de conteneurs Docker pour faciliter le développement (bases de données, backends, front), expérimentation avec des assistants IA, scripts pour automatiser certaines tâches (backups, nettoyage, etc.).

Architecture : Fichiers Docker Compose orchestrant plusieurs services, scripts de gestion, configuration d’outils IA accessibles en local, documentation de la stack pour répliquer facilement l’environnement.
